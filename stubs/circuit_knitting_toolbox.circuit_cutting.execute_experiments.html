circuit_knitting_toolbox.circuit_cutting.execute_experiments¶

execute_experiments(circuits, subobservables, num_samples, samplers)[source]¶

Generate the sampled circuits, append the observables, and run the sub-experiments.

Parameters:

  • circuits (QuantumCircuit | dict[str | int, QuantumCircuit]) – The circuit(s) resulting from decomposing nonlocal gates

  • subobservables (PauliList | dict[str | int, PauliList]) – The subobservable(s) corresponding to the circuit(s). If a QuantumCircuit is submitted for the circuits argument, a PauliList is expected; otherwise, a mapping from partition label to subobservables is expected.

  • num_samples (int) – The number of samples to draw from the quasiprobability distribution

  • samplers (BaseSampler | dict[str | int, BaseSampler]) – Sampler(s) on which to run the sub-experiments.

Return type:

CuttingExperimentResults

Returns:

  • A 3D list of length-2 tuples holding the quasi-distributions and QPD bit information for each sub-experiment. The shape of the list is: (num_unique_samples, num_partitions, num_commuting_observ_groups)

  • Coefficients corresponding to each unique subexperiment’s sampling frequency

Raises:

  • ValueError – The number of requested samples must be positive.

  • ValueError – The types of circuits and subobservables arguments are incompatible.

  • ValueError – SingleQubitQPDGates are not supported in unseparable circuits.

  • ValueError – The keys for the input dictionaries are not equivalent.

  • ValueError – The input circuits may not contain any classical registers or bits.
